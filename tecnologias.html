Tecnologías

En esta sección detallo que lenguajes y frameworks componen mi stack tecnológico, aplicado a mis Páginas Web. Mis puntos fuertes son el diseño Front-End responsivo de páginas y aplicaciones web, trabajar con APIs RESTful, frameworks como Next.js y una fuerte inclinación por el detalle.

HTML 5

La realidad es que muchos no lo consideran un lenguaje de programación, y la verdad, es que tienen razón. Sin embargo, la dificultad del mismo no se encuentra en la lógica, si no en la implementacion de una página efectiva, con un SEO optimizado y que realmente pueda hablar con el motor de búsqueda de un navegador. Por suerte, desde que empecé a aprender desarrollo web, me interioricé con HTML5, el desarrollo semántico y el enfoque por un buen SEO y estructura de código ordenada.

CSS 3

Desde que centré mi primer div me di cuenta de que es una árdua tarea maquetar una página, y ni hablar de hacerlo como los profesionales. Sin embargo, gracias al curso de FreeCodeCamp, pude practicar hasta el hartazgo para aprender de manera eficiente el modelo flex-box de CSS y, de esta manera, pude aprender a maquetar páginas como la que estas visitando ahora mismo. Además, aunque al principio sea abrumador, la flexibilidad para modelar y diseñar en este lenguaje es casi infinita, asi que me hace sentir como un pintor con una hoja en blanco en su mano.

JavaScript

Que decir de uno de los mejores lenguajes de programación del mundo (o al menos de los mas usados...). Como mencioné en los otros apartados, dedique gran parte de mi carrera a aprender lenguajes como C, C++ o Java, por lo que estoy enfocandome en el aspecto visual de las páginas actualmente. Sin embargo, sé que Javascript me da la flexibilidad que necesito como desarrollador Front End para hacer páginas interactivas y dinámicas, asi que me encuentro perfeccionando mis habilidades con el lenguaje desde hace un año.

Tailwind CSS

Una vez que dominé CSS, decidí aprender algún framework del lenguaje que me permitiera maquetar mas fácilmente y con menos margen de error. Haciendo un poco de research, decidí que el que mas se adaptaba a mis necesidades (a pesar de haber probado Bootstrap), era Tailwind, asi que decidí enfocarme en este framework. Construí un par de páginas simples y mi primer tienda web hecha con Tailwind y Javascript Podes echarle un vistazo acá.

Git / GitHub

Aprendí estos software de control de versiones en la facultad y entiendo porque tienen la magnitud que tienen. Siento que es la herramienta ideal para cualquier desarrollador de software, y cada dia descubro mas cosas sobre el mismo que me fascinan. Sin duda una de las mejores herramientas (por no decir la mejor) de control de versiones.

Node.js

Previo a aprender React me recomendaron que conozca algo de Node ya que pertenece al ecosistema de JavaScript y su lógica de módulos se aplica en muchísimas aplicaciones. Lo que encontré fue increíble. Una herramienta como Node deberían conocerla todos: su interacción con el S.O., la red y su sistema de archivos son unas de las tantas cosas que ofrece esta plataforma para JavaScript.

Material UI

Decidí dar un paso mas y aprender una biblioteca de componentes para trabajar con React de forma mas rápida, sencilla y compacta. Tengo conocimientos en Material UI, integración o modificación de componentes y diseño responsivo.

React

Como desarrollador Front-End, decidí aprender React como mi fuerte principal, puesto que tiene todo lo que necesito y más para hacer aplicaciones y páginas web de una forma moderna, simple y rápida. En conjunto con Tailwind y Material UI, pude diseñar y desarollar páginas totalmente funcionales y con datos externos sin la necesidad de Back-End ni bases de datos, lo que demuestra la potencia de esta biblioteca.
